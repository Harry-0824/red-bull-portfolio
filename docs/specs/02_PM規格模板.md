## 7. Test Cases

### 功能測試

- [ ] 一般路徑
- [ ] 邊界情境
- [ ] 錯誤輸入
- [ ] 回歸測試

### 建議測試內容

- UI 是否正常顯示
- 表單驗證是否正確
- API 失敗時是否有 fallback
- 響應式是否正確
- build 是否通過
- lint / type check / test 是否通過

---

## 8. Risks

### 可能風險

- 影響既有元件共用邏輯
- 影響樣式一致性
- 影響 SEO / metadata
- 影響部署環境
- 造成未預期 regression

### 緩解方式

- 小步提交
- 先在 worktree 中隔離開發
- 限制改動範圍
- 增加回歸檢查項
