# Task

Portfolio Home Hero V1

## Context

本專案為個人作品集網站，目標是建立可作為後續 AI 工作流驗證樣板的首頁 Hero 區塊。
目前需要先完成首頁第一屏的內容與結構優化，讓首頁在進站時能清楚傳達個人定位、代表作品方向與主要 CTA。

## Goal

優化首頁 Hero 區塊，使其具備以下目的：

1. 清楚展示個人身份與核心定位
2. 提供明確的主要行動按鈕
3. 建立較完整的視覺層級
4. 作為後續 Navbar、Projects、About 區塊優化的基準樣式

## Scope

本次任務僅處理首頁 Hero 區塊，包含：

- Hero 區塊文案結構調整
- Hero 區塊版面階層調整
- CTA 區塊設計
- 基本響應式表現優化
- 不破壞現有整體專案結構前提下，整理 Hero 相關元件

## Non-goals

本次任務不包含：

- 全站重新設計
- Projects 區塊改版
- About / Contact 區塊調整
- API 串接
- 動畫庫導入
- SEO 全站優化
- 新增額外第三方 UI 套件

## UX / UI Notes

Hero 區塊建議至少包含：

- 主標題：說明你是誰
- 副標題：說明你的方向與專長
- 補充描述：說明你目前聚焦的專案類型
- 主要 CTA：查看作品
- 次要 CTA：聯絡我 / 關於我
- 可選補充資訊：技術棧標籤、代表專案提示、個人標語

視覺方向要求：

- 層級清楚
- 可讀性高
- 保持簡潔
- 適合桌機與手機閱讀
- 不追求過度複雜動畫

## Technical Constraints

- 以現有專案技術棧為基礎實作
- 優先沿用現有樣式系統與專案結構
- 不新增不必要依賴
- 保持 TypeScript 型別正確
- 元件命名需清楚
- 樣式修改需避免影響其他頁面區塊
- 若需拆元件，應維持可重用性

## Acceptance Criteria

- 首頁 Hero 區塊完成明確的標題、副標題、描述與 CTA
- 桌機版具有清楚的視覺層級與留白
- 手機版不會出現明顯跑版、截斷或擁擠問題
- 至少提供一個主要 CTA 與一個次要 CTA
- 修改後專案可正常執行與 build
- 不引入新的版面錯誤
- 不修改本次範圍外的大量內容

## Test Cases

1. 桌機寬度下開啟首頁，確認 Hero 區塊內容完整可讀
2. 手機寬度下開啟首頁，確認標題、描述與按鈕排列正常
3. 點擊 CTA，確認連結目標正確
4. 執行 lint / build，確認未新增明顯錯誤
5. 檢查本次修改未破壞其他主要區塊

## Risks

- 現有首頁結構若耦合過高，Hero 調整可能影響其他區塊
- 若目前樣式結構不清楚，可能需要先做少量整理
- 若文案未先定義，Copilot 可能產出過度 generic 的內容

## Done Definition

當以下條件成立時，視為完成：

- Hero 區塊完成第一版優化
- 通過基本人工驗收
- 變更檔案可被清楚說明
- 本次任務可寫回 Tasks 與 Learnings
